## 一、测试目标

这篇长文的目标很简单：让页面里有足够多的标题层级和滚动距离，方便检查右侧目录是否能够跟随当前阅读位置高亮，目录点击后是否能够准确跳转到对应段落，以及右下角上方的返回顶部按钮是否可以顺畅工作。

### 1.1 目录栏检查点

需要检查的重点包括：

1. 右侧目录是否默认可见。
2. 点击目录任一标题后，页面是否平滑滚动到对应位置。
3. 当前阅读段落变化时，目录高亮是否会同步切换。
4. 标题包含中文时，锚点是否仍然可用。

### 1.2 返回顶部检查点

返回顶部按钮需要在页面滚动一定距离后出现，并且点击以后直接回到文章开头附近，而不是停在中途或者毫无反应。

## 五、数学公式测试

文章页必须保证行内公式和块级公式都能正常渲染，不然技术文章体验会很差。

### 5.1 行内公式

牛顿第二定律可以写成 $F = ma$，质能方程可以写成 $E = mc^2$，而毕达哥拉斯定理可以写成 $a^2 + b^2 = c^2$。

### 5.2 积分与求和

下面是一个常见的块级数学公式：

$$
\int_0^1 x^2 \, dx = \frac{1}{3}
$$

再补一个求和公式：

$$
\sum_{n=1}^{10} n = 55
$$

### 5.3 矩阵与极限

$$
A =
\begin{bmatrix}
1 & 2 & 3 \\
0 & 1 & 4 \\
5 & 6 & 0
\end{bmatrix}
$$

$$
\lim_{x \to 0} \frac{\sin x}{x} = 1
$$

```js
const sections = Array.from(document.querySelectorAll('#post-content h2, #post-content h3'));

function activeHeading(scrollTop) {
  return sections.findLast((section) => section.offsetTop - 120 <= scrollTop);
}

window.addEventListener('scroll', () => {
  const current = activeHeading(window.scrollY);
  console.log('current heading:', current ? current.textContent : 'none');
});
```

## 八、长段落滚动测试

这一部分故意写长一点，让页面出现明显滚动距离，用来测试右侧目录高亮切换和返回顶部按钮显示逻辑。

### 8.1 第一段长文本

当一篇文章真的写得比较长时，读者很少会只从头看到尾。更常见的行为是先扫一眼目录，再跳到自己关心的标题位置。所以右侧目录不是“有没有都行”的装饰，而是长文阅读体验的一部分。目录必须在标题切换时更新高亮，也必须允许用户通过点击目录快速定位。

### 8.2 第二段长文本

返回顶部按钮也是类似的道理。文章越长，按钮的存在感越强。如果按钮位置不稳定、点击后无效、或者返回的位置不准确，体验会非常割裂。尤其是当页面右下角还有立绘、归档按钮等固定元素时，返回顶部按钮需要在视觉和交互上都避让得足够清楚。

### 8.3 第三段长文本

对于技术博客来说，标题层级往往非常细。除了二级标题，还会频繁出现三级标题、四级标题。如果目录只能展示大标题，或者小标题不能点击，读者在查资料时会非常难受。所以这篇测试文章故意加入了多个层级，确保目录树本身也是可以验证的。

### 8.4 第四段长文本

Markdown 内容渲染也不能只看纯文字。图片、动图、视频、公式、表格、代码块，这些内容通常会一起出现在一篇文章中。只要其中任何一种元素的样式不稳定，就会影响整篇文章的观感。因此这篇文章更像一份页面自检清单，而不是普通随笔。

## 九、结语

如果你现在看到这篇文章右侧目录可以正常出现、点击后可以切换到对应标题、页面往下滚动后返回顶部按钮会出现在右下角立绘正上方，并且图片、GIF、视频、数学公式和化学式都能显示出来，那就说明这次文章页的关键交互已经基本修好了。
